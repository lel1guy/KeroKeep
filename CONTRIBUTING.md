# Contributing to Kero Keep

Thanks for checking out the repo. Kero Keep is a source-available solo project — the code is public so you can learn from it, but I'm the primary developer.

## What I Welcome

| Contribution | How |
|-------------|-----|
| 🐛 **Bug reports** | Open an issue. Include: what happened, what you expected, steps to reproduce. |
| 💡 **Feature ideas** | Open an issue. Tag it `enhancement`. I read everything. |
| 🔧 **Code PRs** | Small, focused changes. Bug fixes, performance improvements, typos. Discuss big changes in an issue first. |
| 📖 **Documentation fixes** | Typos, broken links, unclear sections — always welcome. |

## What I Don't Accept

- Large feature PRs without prior discussion
- Code that changes the license or license headers
- Art/sound assets (I'll commission or create those)

## Before You PR

1. Open an issue describing the change (unless it's a one-line fix)
2. Build and test locally on Godot 4 Mono with .NET 8
3. Keep changes focused — one PR, one problem
4. Follow the existing code style (PascalCase, `KeroKeep` namespace)
